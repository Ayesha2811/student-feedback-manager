VersionForge – Student Feedback Manager

Department of Computer Applications, PES University

1. Scenario

Your team is hired by a startup named EduTrack, which builds educational tools. You’ve been asked to develop a Student Feedback Manager, maintain documentation, automate tests, and follow version control practices professionally.

Hint: Use Git branches to handle changes from multiple team members and features.

2. Project Requirements (Broken by Version)

Version 1.0.0

Features:
  • feedback_entry.py: Collect student feedback.
  • score_calculator.py: Compute average score.
Other Tasks:
  • Write at least 2 unit tests using pytest.
  • Create GitHub Actions workflow for testing.
  • Push initial project structure with documentation.

Version 1.0.1 (Given at 55th minute)

Feature Request: Add a function that summarises all feedback (e.g., show top scores and grade-wise counts).
  • Create a new branch feature/summary.
  • Add feedback_summary.py.
  • Update documentation and test cases.
  • Merge to main using a PR.

Version 1.1.0 (Given at 85th minute)

Feature Request: Export feedback to a .txt file.
  • Add a new file report_generator.py.
  • Create GitHub Issue + Milestone titled “v1.1 Release”.
  • Link Issue to Milestone and document the change.

Version 1.2.0 (Given at 115th minute)

Final Feature Request:
  • search_feedback.py: Search feedback by student name.
  • count_feedback.py: Count total entries submitted.
  • Include new tests and version entry in CHANGELOG.md.

Hint: Modularise code into files and maintain changelogs. Use branches and merge wisely.

3. Required Project Artifacts

  • Python files for each feature
  • Test scripts (minimum 1 per feature)
  • README.md, LICENSE, CONTRIBUTING.md, CODE_OF_CONDUCT.md, CHANGELOG.md
  • .github/workflows/ directory with one test automation YAML
  • At least 3 GitHub Issues (linked to 1 Milestone)
  • One Wiki page describing project usage
  • A TEAM_LOG.md describing team contributions

4. Submission Instructions

  • Make your repository public before the deadline.
  • Ensure that every required file and feature is committed.
  • Submit your GitHub repository link.
  • Final submission time: 3 hours from start.
